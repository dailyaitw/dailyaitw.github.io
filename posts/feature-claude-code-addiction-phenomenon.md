# 當 Claude Code 成為新型「癮」：開發者為何停不下來？

## 從矽谷到全球，一場靜默的成癮浪潮

2026 年初，一個奇特的現象正在科技圈蔓延：越來越多開發者公開承認自己「上癮」了——不是遊戲、不是社群媒體，而是 Anthropic 的 AI 程式設計工具 Claude Code。

這不是誇張的比喻。Y Combinator 執行長 Garry Tan 在社群媒體上寫道：**「對 Claude Code 上癮到，昨天連續工作了 19 個小時，凌晨 5 點才睡。」** 在 2026 年 SXSW 大會上，他更進一步描述這種狀態為「cyber psychosis（網路精神錯亂）」——每晚只睡四小時，不是因為工作壓力，而是因為太興奮而停不下來。

---

## 影響力人士的第一手證言

### Garry Tan（Y Combinator 執行長）

Tan 不只是口頭說說。他在 3 月 12 日公開分享了自己的 Claude Code 設定檔——一個名為「gstack」的自定義配置。這條推文迅速爆紅：GitHub 倉庫在短時間內累積了近 20,000 顆星、被 fork 超過 2,200 次，並登上 Product Hunt 熱門。他在推文中寫道：**「感謝 @bcherny 做出了人們想要的東西。我對程式設計的熱愛感謝你，我的睡眠分數就不感謝了。」**

### Andrej Karpathy（前 Tesla AI 總監、OpenAI 共同創辦人）

Karpathy 稱這是**「20 年程式設計生涯中，最大的工作流程轉變。」** 2025 年 11 月，他的工作還是 80% 手動寫程式、20% AI 輔助；到了 12 月，直接反轉為 80% AI、20% 手動修補。他在 X 上分享了連續數週密集使用 Claude Code 的筆記，坦承人類已不再是程式碼的主要撰寫者，而是指揮者、監督者和協調者。

但 Karpathy 也發出警告：他預言 2026 年將是「slopacolypse（爛碼末日）」——GitHub、Substack、arXiv、X/Instagram 以及所有數位媒體上，都將充斥著 AI 生成但品質堪憂的內容。

### Boris Cherny（Claude Code 創造者、Anthropic 工程主管）

Claude Code 的創造者本人透露，他個人已經**「連續兩個多月 100% 由 AI 撰寫程式碼，連小修改都不手動做了。」** Claude Code 團隊的開發工作幾乎 100% 由 Claude Code 搭配 Opus 4.5 完成。

在與 Karpathy 的對談中，Cherny 預測 2026 年底，**「每個人都會是產品經理、每個人都會寫程式，『軟體工程師』這個頭銜將開始消失。」** 他對「爛碼末日」的回應是技術性的樂觀——在 Anthropic 內部，每個 PR 都會開一個新的上下文窗口，讓 Claude 審查 Claude 寫的程式碼。

### Malte Ubl（Vercel 技術長）

Ubl 在假期中每天花 10 小時用 Claude Code 開發新軟體，完成了一個原本需要一年的專案。他形容這種體驗：**「每次運行帶來的多巴胺衝擊，就像在拉斯維加斯玩吃角子老虎一樣讓人上癮。」**

### Daniel Miessler（知名資安研究者、作者）

Miessler 在 2026 年 1 月 8 日發表了一篇極具影響力的文章《Claude Code Addiction is Addiction to Creation》，提出了一個重新定義「上癮」本質的觀點：**「這不是對 Claude Code 的上癮，而是對『創造』本身的上癮。」** 他認為，程式設計者現在能做到 5 倍、10 倍、甚至 100 倍於一年前的產出；過去有想法卻不會寫程式的人，現在也能做到差不多的事情。即使 Claude Code 的上癮是真的，**「那也是對色情、Netflix 或 TikTok 的升級。」**

## Bloomberg 的警告：「2026 年大生產力恐慌」

Bloomberg 在 2026 年 2 月的封面報導中，將這個現象命名為**「The Great Productivity Panic of 2026（2026 年大生產力恐慌）」**。報導指出，AI 程式設計代理人原本承諾讓軟體開發變得更容易，結果卻引發了一場不惜一切代價拼速度的高壓競賽。

資深工程師 Steve Yegge 在 Medium 上發表了題為《The AI Vampire》的文章，將 AI 比喻為 FX 影集《乃出個乾坤》（What We Do In The Shadows）中的「能量吸血鬼」Colin Robinson——它不是靠命令你加班來榨乾你，而是靠讓你「想要」繼續做下去。

Yegge 寫道：**「新型過勞不是你的老闆要求你加班。而是 Claude 問：『這個專案還有什麼需要我做的嗎？』然後你說好、好、好——因為很好玩、因為很有效率、因為 AI 是你的夥伴，不是你的雇主。」**

加州大學柏克萊分校的一項為期八個月、涵蓋 200 人美國科技公司的研究也證實：AI 工具實際上是**加重**了工作量，導致更多倦怠，長期來看反而拖累了整體工作效率。

## 數據背後的現象

- **73% 的工程團隊**每天使用 AI 程式設計工具（2026 年），較 2025 年的 41%、2024 年的 18% 大幅增長
- 社群平台上出現了新詞彙**「Claude-pilled」**，意指完全沉迷於 Claude 無法自拔
- 文化現象級用語如**「Claudeholism: The New Addiction of the Century（Claude 成癮：新世紀的癮症）」**開始流傳
- Fortune 報導：OpenAI 和 Anthropic 的工程師表示 AI 現在撰寫他們 **100% 的程式碼**
- Scientific American 專題報導 Claude Code 如何將「vibe coding（氛圍程式設計）」帶給每一個人

## Karpathy 與 Cherny 的終極對話

在一場被 36kr 稱為「程式設計之死？」的對談中，Karpathy 和 Cherny 針對 AI 取代人類寫程式碼的現實進行了坦率交流：

**Karpathy 的立場**——謹慎但承認不可逆：
> 「人類正站在『大腦萎縮』與『能力進化』的十字路口。」

**Cherny 的立場**——技術性樂觀：
> 「爛碼末日不會來。AI 審查 AI——在 Anthropic，每個 PR 都由 Claude 審查 Claude 的程式碼。」

**共識**：
> 「2026 年是產業消化新能力、發生相變的關鍵年。」

## 台灣觀點：我們該怎麼看？

1. **「上癮」的本質是生產力的量子躍遷**：當一個工具能讓你在一週內完成過去一年的工作量，「停不下來」不是病態，而是理性反應。但 Berkeley 的研究提醒我們：倦怠是真實的，節制是必要的。

2. **台灣開發者正在錯過這波浪潮嗎？** 73% 的全球工程團隊已每天使用 AI 程式設計工具，但台灣的採用率仍明顯落後。語言障礙（Claude Code 的 CLI 介面全英文）和企業文化（對 AI 生成程式碼的信任度低）是兩大阻力。

3. **「軟體工程師」頭銜的消亡對台灣意味著什麼？** Cherny 預測這個頭銜年底將開始消失。台灣的半導體產業可能有天然護城河，但純軟體開發者需要重新定位——從「寫程式碼的人」轉型為「指揮 AI 的人」。

4. **Daniel Miessler 的框架最值得借鏡**：與其恐懼「上癮」，不如將它重新定義為「對創造的渴望被釋放」。當進入門檻消失，真正的差異化不再是寫程式碼的能力，而是「知道該創造什麼」的能力。

5. **Bloomberg 的「大生產力恐慌」不是遠在天邊的故事**：台灣的科技公司如果還在用傳統的「產出量」衡量開發者績效，很快就會面臨 AI 輔助開發者與傳統開發者之間 10 倍以上的生產力差距。管理層需要重新定義什麼是「好的工作」。
